Enhancement 1: Software Engineering and Design

This enhancement focused on improving the structure, security, and maintainability of the inventory application. While the original application had working inventory and login features, the code needed to be better organized into separate responsibilities. The enhanced version reorganized the project into a Model-View-ViewModel structure. This helped separate the user interface from the application logic. Additionally, I improved the authentication process. Originally, passwords were stored in plain-text. To fix this, I implemented password hashing and salting to ensure credentials were not being exposed. A hashed password and a unique salt are stored for each user. Additional tests were added to verify login behavior, account creation, duplicate usernames, and database migration.

The most challenging part of this enhancement was trying to ensure that I successfully re-wired all of the functions, while also making sure I did not miss anything. I demonstrated my skills in software design, secure programming practices, and refactoring. This enhancement made the application more organized and easier to maintain.

Enhancement 2: Data Structures and Algorithms

This enhancement focused on improving how data is searched, organized, and displayed. The original application allowed users to view inventory data and had minimal features for managing items, but there was no specific feature that allowed users to find or organize items. Without an added search structure, it would require searching through the inventory list one item at a time. As the number of inventory items increases, searching through the list becomes less efficient because each item may need to be checked before a match is found. This results in O(n) time complexity. To improve this, I implemented a custom HashMap that supports direct item lookup by name. Items are stored in the HashMap using the item name as the key. As the inventory list grows, the matching item can be retrieved more efficiently. This results in an O(1) average time complexity.

I also implemented a Binary Search Tree (BST) to support alphabetical organization of items. The BST stores items based on name, and an in-order traversal is used to display the items alphabetically. This traversal has a time complexity of O(n) because every item has to be visited once to show the full list.

The challenge I faced during this enhancement was choosing a data structure that matched the needs of the application. It also became difficult to ensure both features worked well together. The MVVM structure made it much easier to navigate and implement these features. This enhancement allowed me to re-visit Big O notation and expand my knowledge. It demonstrates my ability to implement data structures and evaluate the trade-offs. The HashMap improved the application by making item lookup more efficient, while the BST improved organization by allowing inventory items to be displayed alphabetically. These made the application more usable, efficient, and scalable.

Enhancement 3: Databases

This enhancement focused on improving data retrieval features of the Grazioso Salvare dashboard. The original dashboard was connected to a local MongoDB database, and it allowed users to filter records by rescue types. The filter features remain useful; however, the original search function only allowed exact record matches. For example, if a user wanted to find a dog named Rocky, they would have to type the full name.

To improve this artifact, I migrated the database connection to MongoDB Atlas and added Atlas Search functionality. This ensured that the dashboard could support fuzzy search, partial search, and relevant searches across multiple fields. These fields include breed, color, and name. This enhancement made the dashboard more flexible, and it improves user experience by offering advanced and efficient search options. This is especially useful for searching through large datasets.

The largest obstacle I faced when working on this enhancement was figuring out how to implement advanced features into an already existing application. Another challenge was trying to figure out the best way to present this enhancement to my professor without exposing sensitive information and still allowing them to view the dashboard on their end. This enhancement allowed me to build experience in areas that were new to me. I had never migrated a project from local MongoDB to MongoDB Atlas before. Through extensive research, I was able to successfully create a cloud database connection, import the dataset, configure access, and update the dashboard to retrieve data from Atlas. This demonstrates my ability to learn through outside resources and trial and error with minimal guidance. It helped me grow in cloud integration, secure credential handling, and implementing data-driven tools that better support user needs.
